# VentureOS Case Screen

## Purpose

The Case screen is the primary working surface of VentureOS.

VentureOS is not centred on chat.  
It is centred on Cases.

A Case represents a venture, project, investigation, architectural decision, or strategic question being worked through the VentureOS lifecycle.

The Case screen should allow a user to understand:

- what the Case is
- why it exists
- what is currently known
- what remains uncertain
- what has been decided
- what objections exist
- what should happen next

## Design Principle

The Case screen should answer one question:

> What is the current state of this Case, and what is the next responsible action?

## Initial Layout

The initial Case screen should contain the following sections.

---

## 1. Case Header

Displays the basic identity of the Case.

Fields:

- Case ID
- Title
- Status
- Phase
- Owner
- Created date
- Updated date
- Confidence
- Tags

Purpose:

Give immediate context.

Example:

```text
Case 000001 — VentureOS
Status: Discovery
Phase: Foundation
